#### 基础

##### 如何访问用户文件夹（User Folder）

访问该文件夹最简单的方法是通过  
`BeamNG.drive 启动器（Launcher）` → `管理用户文件夹（Manage User Folder）` → `在资源管理器中打开（Open in Explorer）`

##### 工作原理

BeamNG.drive 的文件夹结构由两个主要文件夹组成：

- **安装文件夹（Install Folder）**：程序安装所在的位置
- **用户文件夹（User Folder）**：用户自定义文件所在的位置

用户文件夹通常包含以下内容：

| **文件** | 说明 | 文件夹 |
| --- | --- | --- |
| **Cache** | 游戏缓存的所有文件，用于加快加载 | `/temp` |
| **Mods** | 包含所有模组（含仓库下载） | `/mods` |
| **Replays** | 游戏内录制并保存的回放 | `/replays` |
| **Screenshots** | 游戏内拍摄的截图 | `/screenshots` |
| **Settings** | 所有用户设置，包括控制设置 | `/settings` |
| **车辆配置（Vehicle Configurations）** | 用户创建的车辆配置 | `/vehicles/..` |
| **世界编辑器（World Editor）** | 在世界编辑器中保存的内容 | 与关卡相关的保存位于 `/levels/..`，*Flowgraph* 保存位于 `/flowEditor` |

这种分离方式允许在不影响原始安装的情况下进行修改，从而可以在任何时候轻松恢复到原始状态。

---

##### 结构与优先级

**用户文件夹（User Folder）** 的结构与 **游戏文件夹（Game Folder）** 相同。如果在两个文件夹的相同*相对路径*下都存在某个文件，则会优先加载 **用户文件夹** 中的版本，而不是原始文件。

**不建议** 将用户文件夹存放在与安装文件夹相同的位置。

如果你使用的是 Steam，它可能会在文件验证、更新或重新安装过程中检测到这些未知文件并将其删除。这将导致你在没有任何警告的情况下丢失所有模组、设置以及自定义内容。

#### 用户文件夹变更（> 0.22）

从游戏版本 **0.22.0.0** 开始，**用户文件夹** 发生了一些重要变化，包括新的默认位置以及其工作方式。

##### 版本化与数据迁移

在 **0.22.0.0** 之前的版本中，**用户文件夹** 在不同的程序版本之间是共享的。

对所有程序版本复用同一个用户文件夹，在用户进行过修改的情况下可能会引入问题。例如，用户如果使用过易于访问的世界编辑器（World Editor），这些修改在直接用于另一个程序版本时，可能会产生冲突并破坏内容。

为了降低此类问题的风险，**用户文件夹** 中的数据引入了版本化机制：

```
Userfolder/BeamNG.drive/0.22/<files>
```

当检测到新的*程序版本*时，**游戏启动器（Launcher）** 会负责将部分文件从旧版本迁移到新版本中。

当检测到新版本时的启动器提示信息

自动迁移的数据包括：

| 文件夹 | 说明 |
| --- | --- |
| `/mods` | 包含所有模组（包括仓库下载的模组） |
| `/replays` | 游戏内录制并保存的回放 |
| `/screenshots` | 游戏内拍摄的截图 |
| `/settings` | 用户设置，包括控制设置 |
| `/trackEditor` | 与赛道编辑器（Track Builder）相关的文件 |
| `/vehicles` | 仅限用户创建的车辆配置（`*.pc` 及其对应的 `*.jpg` 文件） |

如有需要，旧版本中的其他文件也可以手动迁移。

---

##### 手动数据迁移

如果由于某些原因，**数据迁移** 过程未能成功完成，你可以按照以下步骤进行**手动数据迁移**：

1. 打开*旧的用户文件夹位置*（旧的默认路径为 `My Documents\BeamNG.Drive`）
2. 通过 `启动器 → 管理用户文件夹 → 在资源管理器中打开` 导航到*新的位置*，并**打开当前程序版本对应的文件夹**
3. 根据需要，将文件从*旧文件夹*复制到*新文件夹*
